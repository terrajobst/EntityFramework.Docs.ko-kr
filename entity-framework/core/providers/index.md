## <a name="adding-a-database-provider-to-your-application"></a><span data-ttu-id="af567-220">애플리케이션에 데이터베이스 공급자 추가</span><span class="sxs-lookup"><span data-stu-id="af567-220">Adding a database provider to your application</span></span>

<span data-ttu-id="af567-221">대부분의 EF Core용 데이터베이스 공급자는 NuGet 패키지로 배포됩니다.</span><span class="sxs-lookup"><span data-stu-id="af567-221">Most database providers for EF Core are distributed as NuGet packages.</span></span> <span data-ttu-id="af567-222">즉, 명령줄 도구에서 `dotnet`을 사용하여 설치할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="af567-222">This means they can be installed using the `dotnet` tool in the command line:</span></span>

``` console
dotnet add package provider_package_name
```

<span data-ttu-id="af567-223">또는 Visual Studio에서 NuGet의 패키지 관리자 콘솔을 사용합니다.</span><span class="sxs-lookup"><span data-stu-id="af567-223">Or in Visual Studio, using NuGet's Package Manager Console:</span></span>

``` powershell
install-package provider_package_name
```

<span data-ttu-id="af567-224">기능이 설치되면 종속성 주입 컨테이너를 사용하는 경우 `DbContext`, `OnConfiguring` 메서드 또는 `AddDbContext` 메서드 중 하나에서 공급자를 구성합니다.</span><span class="sxs-lookup"><span data-stu-id="af567-224">Once installed, you will configure the provider in your `DbContext`, either in the `OnConfiguring` method or in the `AddDbContext` method if you are using a dependency injection container.</span></span>
<span data-ttu-id="af567-225">예를 들어 다음 줄에서는 전달된 연결 문자열을 사용하여 SQL Server 공급자를 구성합니다.</span><span class="sxs-lookup"><span data-stu-id="af567-225">For example, the following line configures the SQL Server provider with the passed connection string:</span></span>

``` csharp
    "Server=(localdb)\mssqllocaldb;Database=MyDatabase;Trusted_Connection=True;");
```  

<span data-ttu-id="af567-226">데이터베이스 공급자는 EF Core를 확장하여 특정 데이터베이스에 고유한 기능을 사용할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="af567-226">Database providers can extend EF Core to enable functionality unique to specific databases.</span></span>
<span data-ttu-id="af567-227">개념은 상당 부분 대부분의 데이터베이스에서 공통이며 기본 EF Core 구성 요소에 포함됩니다.</span><span class="sxs-lookup"><span data-stu-id="af567-227">Some concepts are common to most databases, and are included in the primary EF Core components.</span></span>
<span data-ttu-id="af567-228">이러한 개념에는 LINQ에서의 쿼리 표현, 트랜잭션, 데이터베이스에서 로드된 후 개체에 대한 변경 내용 추적 등이 포함됩니다.</span><span class="sxs-lookup"><span data-stu-id="af567-228">Such concepts include expressing queries in LINQ, transactions, and tracking changes to objects once they are loaded from the database.</span></span>
<span data-ttu-id="af567-229">일부 개념은 특정 공급자에게만 적용됩니다.</span><span class="sxs-lookup"><span data-stu-id="af567-229">Some concepts are specific to a particular provider.</span></span>
<span data-ttu-id="af567-230">예를 들어 SQL Server 공급자에서는 [메모리 최적화 테이블을 구성](xref:core/providers/sql-server/memory-optimized-tables)할 수 있습니다(SQL Server 특정 기능).</span><span class="sxs-lookup"><span data-stu-id="af567-230">For example, the SQL Server provider allows you to [configure memory-optimized tables](xref:core/providers/sql-server/memory-optimized-tables) (a feature specific to SQL Server).</span></span>
<span data-ttu-id="af567-231">다른 개념은 공급자 클래스에 특정합니다.</span><span class="sxs-lookup"><span data-stu-id="af567-231">Other concepts are specific to a class of providers.</span></span>
