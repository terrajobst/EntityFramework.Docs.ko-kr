## <a name="adding-a-database-provider-to-your-application"></a><span data-ttu-id="31210-224">애플리케이션에 데이터베이스 공급자 추가</span><span class="sxs-lookup"><span data-stu-id="31210-224">Adding a database provider to your application</span></span>

<span data-ttu-id="31210-225">대부분의 EF Core용 데이터베이스 공급자는 NuGet 패키지로 배포됩니다.</span><span class="sxs-lookup"><span data-stu-id="31210-225">Most database providers for EF Core are distributed as NuGet packages.</span></span> <span data-ttu-id="31210-226">즉, 명령줄 도구에서 `dotnet`을 사용하여 설치할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="31210-226">This means they can be installed using the `dotnet` tool in the command line:</span></span>

install-package provider_package_name

<span data-ttu-id="31210-228">기능이 설치되면 종속성 주입 컨테이너를 사용하는 경우 `DbContext`, `OnConfiguring` 메서드 또는 `AddDbContext` 메서드 중 하나에서 공급자를 구성합니다.</span><span class="sxs-lookup"><span data-stu-id="31210-228">Once installed, you will configure the provider in your `DbContext`, either in the `OnConfiguring` method or in the `AddDbContext` method if you are using a dependency injection container.</span></span>

<span data-ttu-id="31210-230">데이터베이스 공급자는 EF Core를 확장하여 특정 데이터베이스에 고유한 기능을 사용할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="31210-230">Database providers can extend EF Core to enable functionality unique to specific databases.</span></span>

<span data-ttu-id="31210-234">예를 들어 SQL Server 공급자에서는 [메모리 최적화 테이블을 구성](xref:core/providers/sql-server/memory-optimized-tables)할 수 있습니다(SQL Server 특정 기능).</span><span class="sxs-lookup"><span data-stu-id="31210-234">For example, the SQL Server provider allows you to [configure memory-optimized tables](xref:core/providers/sql-server/memory-optimized-tables) (a feature specific to SQL Server).</span></span>
